    2.2.4Add Flexbox to the Header

    First we'll add flexbox to the header so that it can accommodate various screen sizes gracefully. Before flexbox, developers used CSS layout properties like float for that purpose. As mobile devices became more popular, the inadequacies of float became apparent: content would often would break, overlap, or overflow off the screen. But it was the only viable option at the time.

    Flexbox (or the Flexible Box Layout Module) was first introduced in 2009 but wasn't fully supported by browsers until 2017—and was welcomed with open arms by the developer community! Developers finally had a tool to achieve modern layouts on any screen size.

    However, the current Run Buddy site uses floats in the <header>. As you can see in the following animation, that part of the page doesn't resize very well:

    The following animation shows how the header will resize after we add flexbox to it:

    That's much smoother, right? Flexbox creates HTML parent/child (or container/container-item) relationships. The parent keeps track of how much space each child occupies and repositions all of the children accordingly, allowing developers to focus only on the child element's content.

    Whenever a new front-end tool shows up in the development world, the first thing you need to determine is whether you can use it—because front-end technologies only work if browsers support them. The website Can I Use (Links to an external site.) tracks technology adoption among desktop and mobile browsers, to help developers choose new technologies for their applications.

    The following image shows how flexbox fares on Can I Use:

    As you can see, flexbox is widely used across browsers, with a 98% adoption rate. This makes your job a little easier, because you know that most of your users will be able to access and fully experience your site.

    Flexbox is quickly becoming the industry standard, but not every front-end tool is quite so universal. Sometimes we have to exclude potential users so that we can use a front-end tool that isn't adopted by every browser. To get a better sense of this, you can search for other tools and note which browsers do or do not support them.

    How Flexbox Works

    When a container is given a display value of flex, it can create a one-dimensional layout, meaning that it controls the distribution of its content along either a horizontal x-axis (row) or vertical y-axis (column).

    By default, a flexbox is set to be a row. In other words, it attempts to fit all of the child elements on one horizontal line until it reaches the right edge of the parent. At that point, it has to decide whether to make all of the child elements narrower—to accommodate a new sibling on that horizontal line (which often results in squished child elements)—or to allow the parent to overflow, or wrap, its content onto the next line.

    Wrapping is commonly used in conjunction with flexbox containers to calculate smart layouts for screens of various sizes. The following image demonstrates how wrapping can work:

    Another thing flexbox does well is to horizontally or vertically align child elements. For example, it can vertically center HTML elements in a container—something that used to be nearly impossible with CSS. Flexbox can also calculate the free space available and distribute it evenly among child elements.

    Prepare the Header for Flexbox

    Much of this phase of the Run Buddy revamp involves replacing CSS properties with flexbox properties. We'll start updating the <header> element by removing some pieces from the CSS file. Let's go!

    In style.css, remove the display: inline declaration from header h1, so that it resembles the following example:

      header h1 {
    font-weight: bold;
    margin: 0;
    font-size: 36px;
    color: #fce138;
  }

    Then remove the float:right declaration from header nav, as shown in the following example:

      header nav {
    margin: 7px 0;
  }

    Now you can apply flexbox by adding the following CSS property declaration to the header rule:

      display: flex;

    That's all you need to add to get the elements back on the same horizontal line! The header should now look like the following image:

    Note that the display: flex declaration makes the <header> a row by default, so there's no need to explicitly declare it. And as you can see, flexbox evenly distributes the space of the parent element (<header>) among the child elements (the <h1> and <nav> links). Interestingly, both of those children are block-level elements by default—but adding flexbox overrides that default behavior.

    pro tip

    To change a flexbox from a row to a column layout, you can use the flex-direction property with a value of column.

    But wait—did you notice that the navigation items seem to be skewed to the left, closer to the <h1>? You can adjust that with a property called justify-content, as you'll soon learn.

    Justify the Header Elements

    The justify-content property applies to elements with a display property value of flex. It allows you to control spacing between child elements by using the following values:

    Which one of these values is right for our layout? We currently have two child elements, <h1> and <nav>, in the <header>. We want to space them as far apart as possible, with <h1> on the left and <nav> on the right. This dictates that there will be some unused space between them.

    The following image depicts what we need to create:

    To achieve that look, let's add the following declaration to the header CSS rule:

    justify-content: space-between;

    This property declaration puts all of the unused space in the <header> between the two elements, with the space-between value. It determines the unused space by adding up the border-width, margin, padding, and content of each child element. If these don't add up to 100% of the parent, then whatever is left over is considered unused.

    pause

    Which CSS concept are we using when we calculate an element's dimensions by adding up its content, padding, border-width, and margin values?

    The CSS Box Model.

    After implementing space-between, the <header> content looks great. But what happens when the screen gets smaller? Resize your browser window to find out.

    Wrap the Content

    As you probably noticed, the content becomes as small and tight as possible or bursts out the right side of the container. You can solve this problem by using the flex-wrap property to instruct the container to let child elements break onto the next line when when they don't fit on one line.

    You can see flex-wrap in action in the following animation:

    By default, the value of flex-wrap is set to none, so we need to explicitly tell it to wrap. Add the following to the header CSS rule:

    flex-wrap: wrap;

    Now the <header> will expand onto a second line if it runs out of room when we resize the browser window!

    Having accomplished this little bit of magic, be sure to save your progress with Git by using the following commands:

    git add .
git commit -m "add flexbox to header"
git push origin feature/flexbox

    Fantastic! Now that you've added flexbox to the header, you can also implement it in the navigation elements.
